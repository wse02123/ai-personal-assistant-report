AI Automation System

텔레그램으로 대화하는
AI 개인비서 자동화 시스템

메시지, 이미지, 문서, 링크를 채팅창에 보내면 AI가 내용을 이해하고 분류, 요약, 저장, 조회, 수정, 리마인드까지 이어주는 개인 정보 관리 자동화입니다.

한 문장 정의

채팅형 입력창을 개인 데이터베이스와 AI 분석 엔진에 연결한 자동화 비서입니다.

핵심 가치

입력은 메신저처럼 가볍게, 저장은 DB처럼 구조적으로, 다시 찾기는 검색처럼 빠르게.

뭘 만든 건가

이 프로젝트는 텔레그램 봇을 사용자 인터페이스로 쓰고, n8n이 전체 자동화 흐름을 제어하며, AI API와 Google 서비스를 연결해 개인 지식과 파일을 정리하는 시스템입니다.

1. 텔레그램에 정보 전송
2. AI가 제목, 분류, 요약, 키워드 추출
3. Google Sheets에 검색 가능한 인덱스 저장
4. 파일은 Drive에 보관, 기한성 항목은 Calendar와 연결
5. 결과를 텔레그램으로 다시 안내

실제 화면으로 보는 결과

n8n은 자동화 흐름을 실행하고, 텔레그램은 사용자가 결과를 확인하는 대화형 화면이 됩니다.

n8n 워크플로우 노드 구성

텔레그램 입력, AI 분석, Sheets 저장, Drive 보관, Calendar 등록, 리마인드까지 이어지는 전체 자동화 흐름입니다.

텔레그램 실제 대화 화면

사용자는 채팅으로 요청하고, 봇은 처리중, 저장 완료, 중복 확인, 덮어쓰기 결과를 대화형으로 안내합니다.

어떤 걸 할 수 있나

IN

아무 형식으로 저장

텍스트, 이미지, PDF, 문서, 링크를 채팅하듯 보내면 기록으로 변환합니다.

AI

자동 분류와 요약

AI가 제목, 대분류, 소분류, 키워드, 요약, 날짜 정보를 추출합니다.

DB

검색 가능한 인덱스

Google Sheets를 개인 DB처럼 사용해 나중에 다시 찾을 수 있게 합니다.

OK

확인형 자동화

중복, 새 카테고리, 수정 후보는 버튼으로 확인한 뒤 처리합니다.

사용자가 하는 일 시스템이 하는 일 결과
메모나 아이디어를 보낸다 내용을 분석해 제목, 분류, 요약, 키워드 생성 구조화된 기록으로 저장
이미지, 캡처, 문서를 보낸다 파일 정보를 읽고 캡션이 부족하면 추가 설명 요청 파일과 설명을 하나의 항목으로 묶어 보관
전에 저장한 내용을 찾아달라고 한다 키워드와 후보 점수로 관련 항목을 찾음 텔레그램에서 후보 확인 후 내용 또는 파일 수신
기존 항목을 바꾸거나 삭제한다 자연어 요청에서 대상 후보를 찾고 사용자 확인 후 Sheets 갱신 채팅 안에서 기록 관리 완료
기한이 있는 내용을 보낸다 날짜를 추출하고 필요 시 Calendar와 리마인드 흐름에 연결 놓치기 쉬운 항목을 다시 알림

지원 입력 타입과 사용 방식

저장, 수정, 삭제, 조회가 모두 채팅 명령과 버튼 확인으로 이어집니다. 사용자는 별도 관리 화면을 열지 않고 대화창 안에서 주요 기능을 처리합니다.

저장되는 데이터 구조

단순히 메시지 원문만 저장하는 것이 아니라, 나중에 검색·수정·자동화에 쓸 수 있도록 여러 필드를 함께 저장합니다. 그래서 이 시스템은 단순 응답형 챗봇이 아니라 데이터베이스와 연결된 개인 운영 도구로 동작합니다.

record_key시스템이 내부적으로 기록을 구분하는 값

도구별 역할

Telegram
사용자와 시스템이 만나는 채팅 인터페이스입니다. 메시지, 파일, 버튼 클릭을 받습니다.
n8n
전체 흐름의 두뇌입니다. 입력 종류를 구분하고, API를 호출하고, 성공·실패 응답을 보냅니다.
AI API
OpenAI API가 텍스트와 파일 맥락을 분석합니다. Gemini API도 같은 위치에 붙일 수 있는 구조입니다.
Google Sheets
제목, 카테고리, 요약, 키워드, 파일 위치, 상태값을 저장하는 인덱스 DB 역할입니다.
Google Cloud
Sheets, Drive, Calendar API를 쓰기 위한 프로젝트, 권한, 서비스 계정 인증 기반입니다.

핵심 설계 포인트

01

채팅을 입력 폼으로 사용

복잡한 UI 없이 평소 쓰는 메신저를 자동화의 시작점으로 만들었습니다.

02

AI 결과를 데이터화

AI 답변을 문장으로 끝내지 않고, 후속 처리 가능한 JSON 데이터로 변환합니다.

03

확인 필요한 자동화

중복, 수정 후보, 새 카테고리는 사용자의 버튼 선택을 거쳐 데이터 품질을 지킵니다.

핵심 구현 패턴

P

Pending 패턴

파일만 먼저 오고 설명이 나중에 오는 경우, 임시 저장소에 상태를 남겨 두 메시지를 하나의 기록으로 합칩니다.

V

검증 가능한 AI

AI의 분류 결과를 바로 저장하지 않고 카테고리 목록, 중복 후보, 사용자 버튼 확인을 거칩니다.

API

외부 API 권한 설계

Sheets, Drive, Calendar는 각각 권한과 응답 구조가 달라 인증 방식과 실패 처리 메시지를 분리해야 합니다.

결과적으로 가능한 일

개인용 정보 수집함 + AI 분류기 + 자동화 실행기를 하나로 묶은 시스템

이 시스템은 단순한 챗봇이 아니라, 사용자가 보낸 정보를 이해하고, 저장 가능한 형태로 정리하고, 나중에 다시 찾고 고칠 수 있게 만드는 개인 운영 도구입니다.
