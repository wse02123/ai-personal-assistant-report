AI Automation System · Oracle Cloud · mTLS 보안

텔레그램으로 대화하는 AI 개인비서 자동화 시스템

메시지·이미지·문서·링크를 채팅창에 보내면 AI가 내용을 이해하고 분류·요약·저장·달력 등록·리마인드까지 이어주는 개인 정보 관리 자동화입니다.
n8n은 오라클 클라우드 서버에서 상시 가동되며, mTLS 인증서로 접근을 통제합니다.

한 문장 정의

채팅형 입력창을 개인 데이터베이스와 AI 분석 엔진에 연결한 자동화 비서입니다.

핵심 가치

입력은 메신저처럼 가볍게, 저장은 DB처럼 구조적으로, 달력과 리마인드는 자동으로.

뭘 만든 건가

이 프로젝트는 텔레그램 봇을 인터페이스로 쓰고, n8n이 전체 자동화 흐름을 제어하며, AI API와 Google 서비스를 연결해 개인 지식과 파일을 정리하는 시스템입니다. n8n은 오라클 클라우드 Always Free VM에서 24시간 상시 가동됩니다.

1. 텔레그램에 정보 전송 (텍스트·이미지·PDF·링크)
2. AI가 제목, 분류, 요약, 키워드, 날짜·시간 추출
3. Google Sheets에 검색 가능한 인덱스 저장
4. 일정/할일이면 Google Calendar 자동 등록
5. 결과를 텔레그램으로 다시 안내

일정 특화 기능 — Phase 1 캘린더

단순 저장을 넘어, 일정 메시지에서 의미 있는 구조를 추출해 Google Calendar에 정확하게 연결합니다. title 작성 규칙부터 반복 일정까지 세밀하게 설계되어 있습니다.

title suffix 정책

AI가 메시지의 예약 완료 여부를 판단해 title 끝에 suffix를 붙입니다. 날짜·요일 정보는 title에 절대 포함하지 않습니다.

summary 고정 양식

일정/할일 카테고리이면 AI가 자유 텍스트 대신 아래 고정 양식으로 summary를 채웁니다. 정보가 없는 항목은 '-'로 표기하고, 레이블은 생략 불가입니다.

시간: HH:MM~HH:MM  |  장소: 장소명  |  멤버: 이름들  |  비고: 기타

Google Calendar 등록 3가지 모드

시간 지정 이벤트

start_time이 있을 때. 시작·종료 시간을 그대로 등록합니다. 종료 시간이 없으면 1시간 기본값.

dateTime: 2026-06-14T09:00:00+09:00
하루종일 이벤트

start_time이 null일 때. 날짜만 지정해 종일 이벤트로 등록합니다.

date: "2026-06-19"
날짜 범위 이벤트

start_date ≠ due_date일 때. 범위 전체를 하나의 이벤트로 등록합니다.

start: "2026-06-16" → end: "2026-06-17"

반복 일정 & notice

RRULE 기반 반복 설정

AI가 반복 패턴을 감지하면 recurrence 필드를 채웁니다.

매주 화요일 → WEEKLY:TUE
매주 월요일 → WEEKLY:MON
매월 15일 → MONTHLY:15

notice — Telegram 전용 안내

반복 일정이 감지되면 Telegram에만 안내 메시지를 보냅니다. Sheets에는 저장되지 않습니다.

"매주/매월 반복 일정입니다.
구글 캘린더에서 반복 설정을 권장합니다."

전체 연계 도식

오라클 클라우드 VM 위에 n8n이 가동되고, Caddy가 트래픽을 제어합니다. Telegram은 입출력, AI API는 이해와 분류, Google Cloud는 인증과 저장 레이어입니다.

저장되는 데이터 구조

단순 메시지 원문만 저장하는 것이 아니라, 검색·수정·자동화에 쓸 수 있도록 여러 필드를 함께 저장합니다. 일정 특화 필드(start_date, start_time, end_time, recurrence)가 추가되어 Google Calendar와 정밀하게 연동됩니다.

Google Sheets

Google Calendar

Google Cloud
Sheets, Drive, Calendar API를 쓰기 위한 프로젝트, 권한, 서비스 계정 인증 기반입니다.

AI 모델 연계 구조

모든 입력 타입에 gpt-4o-mini 단일 모델을 사용합니다. 모델을 통일함으로써 날짜 계산 정확도와 지시문 준수율을 높였습니다. 프롬프트에는 오늘 날짜 기준으로 이번주/다음주 날짜를 미리 계산한 빠른참조 테이블이 포함됩니다.

flowchart LR INPUT["원본 입력\n텍스트·이미지·PDF·URL"] --> ROUTER["gpt-4o-mini\n입력 타입별 프롬프트 구성"] ROUTER --> O1["텍스트 분석\n분류·요약·날짜·시간 추출"] ROUTER --> O2["Vision / PDF 분석\n파일 내용 이해 + 구조화"] O1 --> JSON["표준 JSON\ntitle, L1, L2, entities\ndue_date, start_date, start_time\nend_time, summary, recurrence"] O2 --> JSON JSON --> VALIDATE["카테고리 검증·중복 체크"] VALIDATE --> SAVE["저장·Calendar·응답 자동화"]

빠른참조 테이블 — 날짜 오류 방지 핵심

프롬프트 첫 줄에 오늘=2026-06-14, 이번주 토=2026-06-14, 다음주 화=2026-06-16... 식으로 모든 요일을 실제 날짜로 미리 계산해 주입합니다. AI는 직접 날짜를 계산하지 않고 이 테이블에서 복사만 합니다. gpt-4.1-nano에서 발생하던 날짜 off-by-one 오류가 이 방법으로 해결되었습니다.

핵심 설계 포인트

01

채팅을 입력 폼으로 사용

복잡한 UI 없이 평소 쓰는 메신저를 자동화의 시작점으로 만들었습니다.

02

AI 결과를 데이터화

AI 답변을 문장으로 끝내지 않고, 후속 처리 가능한 JSON 구조로 변환합니다.

03

확인 필요한 자동화

중복, 수정 후보, 새 카테고리는 버튼 선택을 거쳐 데이터 품질을 지킵니다.

04

서버 상시 가동

Oracle Cloud Always Free VM으로 로컬 PC 의존 없이 24시간 자동화가 동작합니다.

05

mTLS 단말 인증

관리 화면은 클라이언트 인증서 설치 단말만 접근 가능. Telegram 봇은 개방 유지.

06

일정 특화 분석

title suffix, summary 고정 양식, Calendar 3가지 모드를 통해 일정 데이터 품질을 높였습니다.

핵심 구현 패턴

P

Pending 패턴

파일만 먼저 오고 설명이 나중에 오는 경우, 임시 저장소에 상태를 남겨 두 메시지를 하나의 기록으로 합칩니다.

V

검증 가능한 AI

AI의 분류 결과를 바로 저장하지 않고 카테고리 목록, 중복 후보, 사용자 버튼 확인을 거칩니다.

R

빠른참조 프롬프트

날짜 계산을 AI에게 맡기지 않고, 오늘 기준 모든 요일의 날짜를 프롬프트에 미리 주입해 오류를 방지합니다.

결과적으로 가능한 일

개인용 정보 수집함 + AI 분류기 + 일정 자동화 실행기를 하나로 묶은 시스템

이 시스템은 단순한 챗봇이 아니라, 사용자가 보낸 정보를 이해하고, 저장 가능한 형태로 정리하고, 일정이면 캘린더까지 자동으로 연결하는 개인 운영 도구입니다. Oracle Cloud 서버에서 상시 가동되고, mTLS로 보안이 유지됩니다.
